# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Compressions, please report it responsibly.

**Email:** Open a [GitHub Security Advisory](https://github.com/Ismael-Barajas/compressions/security/advisories/new) (preferred) or email the maintainer directly.

Please include:

- Description of the vulnerability
- Steps to reproduce
- Affected version(s)

You should receive a response within 7 days. Please do not open a public issue for security vulnerabilities.

## Scope

This policy covers the Compressions application code. Vulnerabilities in bundled third-party binaries (FFmpeg, Ghostscript) should be reported to their respective upstream projects.
